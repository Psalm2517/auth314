Pi Sign-in, made simple.

Auth314 handles the Pi Sign-in OAuth flow so you don't have to. Get verified Pi identity delivered straight to your callback URL, no Pi Developer Portal app and no OAuth to build.

View on GitHub

How it works

  1. 1 Your app calls /verify/init
  2. 2 User clicks the link and signs in with Pi
  3. 3 Auth314 delivers the verified identity to your callback URL

Webhook payload

{
  "platform_user_id": "...",
  "guild_id": "...",
  "verified": true
}

Pi identity (uid, username) stays internal to Auth314 and is never included in the webhook payload.
